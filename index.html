Git — система контролю версій

Підготував: Kliuchuk Vadim

Що таке система контролю версій?

Система контролю версій (VCS) — це програмне забезпечення, яке допомагає відстежувати зміни в коді, документах та інших файлах проєкту. Вона дозволяє зберігати історію змін, співпрацювати з командою та повертатися до попередніх версій при необхідності.

Git — це найпопулярніша сучасна система контролю версій, розроблена Лінусом Торвальдсом у 2005 році. Git є розподіленою системою, що забезпечує надійність та високу продуктивність.

Переваги Git

  • Розподілена архітектура — кожен розробник має повну копію репозиторію.
  • Швидке перемикання між версіями та гілками.
  • Можливість паралельної роботи над проєктом у команді.
  • Висока швидкість виконання операцій.
  • Безкоштовний та з відкритим кодом.
  • Широка підтримка онлайн-сервісів, таких як GitHub, GitLab, Bitbucket.

10 базових команд Git

  1. git init — створити новий Git-репозиторій у поточній папці.
  2. git clone [url] — клонувати існуючий репозиторій з віддаленого сервера.
  3. git status — перевірити статус файлів у репозиторії.
  4. git add [файл] — додати зміни у файл до індексу (стейджинг).
  5. git commit -m "повідомлення" — зафіксувати зміни з коментарем.
  6. git push — відправити зміни у віддалений репозиторій.
  7. git pull — отримати зміни з віддаленого репозиторію та об’єднати з локальними.
  8. git log — перегляд історії комітів.
  9. git branch — перегляд або створення нових гілок.
  10. git merge [гілка] — об’єднати іншу гілку з поточною.
